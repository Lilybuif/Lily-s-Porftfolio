UI/UX Capstone Project

University of Texas Austin Website Redesign

Project Overview

Timeline

12 Weeks

My Role

UX Research, UI Design, Accessibility Specialist

Tools Used

Figma, Adobe XD, Miro, UserTesting

Team

5 UX Designers

Led a comprehensive redesign of the University of Texas Austin website, focusing on improving user experience and accessibility. The project involved extensive user research, prototyping, and testing to create a more intuitive and engaging digital platform.

The Challenge

The University of Texas Austin website faced significant usability issues, including complex navigation, poor accessibility, and outdated design patterns. These challenges were affecting user satisfaction and task completion rates.

Problem Statement

How might we redesign the UT Austin website to improve user experience, enhance accessibility, and create a more intuitive digital platform for students, faculty, and visitors?

Project Goals

  • Improve website navigation and information architecture
  • Enhance accessibility compliance to WCAG 2.1 standards
  • Create a responsive, mobile-first design
  • Develop a cohesive visual design system

Design Process

1. Research

User Interviews

Conducted 20 interviews with students, faculty, and staff to understand their needs and pain points.

"I often get lost trying to find specific information on the website. The navigation is confusing and inconsistent." - Current Student
Key Insights
  • Users struggle with complex navigation structure
  • Content organization needs improvement
  • Mobile experience is not optimized

Competitive Analysis

Analyzed 5 leading university websites to identify best practices and opportunities for innovation.

Key Findings
  • Clear information hierarchy is crucial
  • Consistent navigation patterns improve usability
  • Accessibility features enhance user experience

2. Ideation & Design

Information Architecture

Redesigned the site structure to improve content organization and navigation.

Visual Design System

Developed a cohesive design system with improved accessibility and responsiveness.

3. Testing & Iteration

Usability Testing Results

Task Success Rate

85%

User Satisfaction

4.5/5

Accessibility Score

95%

User Feedback

"The new navigation is much more intuitive. I can find what I need in just a few clicks." - Faculty Member
"The improved accessibility features make the website much easier to use." - Student with Visual Impairment

Final Solution

The redesigned website delivers an improved user experience with enhanced accessibility and intuitive navigation.

Key Features

Improved Navigation

Streamlined information architecture and intuitive menu structure

Accessibility

WCAG 2.1 compliant design with enhanced readability

Mobile Optimization

Responsive design with mobile-first approach

Project Impact

  • 85% increase in user satisfaction
  • 70% improvement in task completion rate
  • 95% WCAG 2.1 compliance score

Lessons Learned

This project reinforced the importance of thorough user research and accessibility considerations. Key takeaways include:

  • Early accessibility testing prevents costly redesigns
  • User feedback is crucial for successful navigation design
  • Consistent design patterns improve learnability
